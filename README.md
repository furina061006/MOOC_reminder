# MOOC Reminder 

> 中国大学MOOC 作业提醒助手 — 再也不会漏交作业啦！！！
> hiahiahia b(￣▽￣)d　

> <span style="color:red">注:</span>
> 本插件 99.99% 的 代码贡献 依靠 Vibe Coding ~(Claude~ ~Code~ ~+~ ~DeepSeek~ ~API)~

打开 [中国大学MOOC](https://www.icourse163.org/) 完成线上作业时，插件自动爬取「测验与作业」和「考试」板块的未完成作业，在扩展图标上显示数量，点击即可查看清单。

---

## 功能一览

| 功能 | 说明 |
|------|------|
| 🔔 **徽章提醒** | 扩展图标上直接显示未完成作业数量 |
| 🔴🟠🔵 **紧急程度** | 红色=已过期 <br> 橙色=48h内截止 <br> 蓝色=正常 |
| 📋 **课程分组** | 按课程分组展示，一目了然 |
| ✅ **自动检测** | 测验类自动检测完成状态<br>~（有得分标记为完成）~ |
| ✋ **手动确认** | 作业类需手动勾选确认，避免误判 |
| 🏷️ **状态标签** | 测验 → `自动检测` <br> 作业 → `手动确认` <br> 互评中 → `互评中` |
| 🌙 **深色模式** | 自动适配系统主题 |
| ⚙️ **可配置设置** | 检查间隔、提醒阈值、自动检测、免打扰时段、每日汇总 |
| 📅 **日历导出** | 一键导出未完成作业为 `.ics` 日历文件 |
| 📝 **手动提醒** | 可手动添加爬虫漏掉或线下作业，支持稍后提醒和课程静音 |
| 📱 **轻量无后台** | 纯浏览器端运行，不占服务器资源 |

## 徽章颜色说明

| 颜色 | 含义 |
|------|------|
| 🟠 橙色 | 有作业在 48 小时内截止 |
| 🔵 蓝色 | 正常状态，无紧急项 |

## 安装方法

### 前提
- Chrome / Edge / 夸克等 **Chromium** 内核浏览器

### 步骤

```bash
# 1. 下载项目代码
git clone <项目地址>  # 或下载 ZIP 解压

# 2. 打开浏览器扩展管理页
chrome://extensions/

# 3. 开启右上角「开发者模式」

# 4. 点击左上角「加载已解压的扩展」

# 5. 选择 MOOC_reminder 文件夹
```

> ⚠️ 本插件未上架 Chrome 网上应用店，需要通过「开发者模式」加载。

## 使用方法

```
1. 登录 icourse163.org
2. 进入任意课程的「测验与作业」或「考试」页面
3. 插件自动爬取 → 扩展图标显示未完成数量
4. 点击扩展图标查看详细清单
5. 测验类 完成后：插件自检测确认完成
   作业类 完成后：勾选复选框确认完成
```

### 使用技巧

- **测验类**（选择题）：插件自动检测得分，完成后`自动检测`
- **作业类**（大题/互评）：需完成后`手动确认`
- **互评中**：互评阶段的作业会显示`互评中`标签
- **不在页面时**：关闭课程页面后，之前爬取的数据仍在 popup 中可查看
- **刷新数据**：打开课程的 测验与作业 页面后，插件每 30 秒自动刷新当前打开的 **所有**课程的 测验与作业；也可点击 popup 的刷新按钮手动刷新

> <span style="color:red">注:</span> 开发期间没赶上考试模块发布作业, 暂时无法确定对于 考试模块 的真实效果

### 筛选功能

popup 左上角可筛选：
- **未完成** — 默认视图，所有未完成项（**含已过期**，与徽章数量一致）
- **已过期** — 只看已过期但仍未完成的作业
- **已完成** — 已勾选/已自动检测完成的作业
- **全部（含过期）** — 所有作业

> 列表按**截止时间从近到远**排序，最紧急的课程分组排在最前。点击任意条目可直接打开对应的作业/课程页面。

## 局限性

- ❌ **需要打开课程页面才能爬取** — 插件通过注入到浏览器页面的脚本运行，关掉页面就无新数据
- ❌ **仅支持 中国大学MOOC** — 不支持学堂在线、超星等其他平台
- ❌ **不跨设备同步** — 数据存在浏览器本地 `chrome.storage.local`，换设备需重新爬取
- ❌ **网页改版可能失效** — 如果 中国大学MOOC 更新页面 DOM 结构，爬虫可能暂时失效（可通过更新 `selectors.json` 修复）
- ❌ **作业类(大题/互评) 无法自动检测完成** — 作业类 需要用户手动勾选确认

## 数据管理

| 操作 | 位置 | 说明 |
|------|------|------|
| ✅ 手动勾选 | popup → 勾选复选框 | 手动标记完成 |
| 🔄 刷新爬取 | popup → 🔄 按钮 | 立即重新爬取当前页面<br>**不会**删除手动标记缓存 |
| 🗑️ 重置数据 | popup → 底部「重置数据」 | 清除所有缓存数据 <br>**会**删除手动标记缓存|
| ⚠️ 崩溃恢复 | 自动 | 遇到数据损坏时弹出恢复界面，可一键清除缓存 |
| ⚙️ 设置 | popup → 右上角齿轮 | 配置后台检查间隔、徽章刷新间隔、截止提醒阈值、自动检测、免打扰时段和每日汇总 |
| 📅 导出日历 | popup → 底部「导出日历」 | 将所有未完成且有截止时间的作业导出为 `.ics`，可导入系统日历 |
| 📝 手动添加 | popup → 筛选栏右侧「+」 | 手动添加爬虫漏掉或线下作业提醒 |
| ⏰ 稍后提醒 | popup → 作业条目时钟按钮 | 24 小时内不再对此条目弹通知（徽章仍计数） |
| 🔕 课程静音 | popup → 课程标题右侧铃铛按钮 | 静音该课程通知/每日汇总（徽章仍计数） |
| 🧹 清除已完成 | popup → 底部「清除已完成」 | 移除已完成记录，未完成作业不受影响 |

## 隐私说明

- 所有数据仅存储在**浏览器本地**
- **不会上传**到任何服务器
- **不收集**任何个人信息
- 仅在 中国大学MOOC 页面运行

## 原理 & 技术栈（给开发者）

### 工作原理

```
用户打开 中国大学MOOC 课程页面
  → Content Script 注入页面
    → 监听 URL 变化（SPA 适配）
    → 找到「测验与作业」「考试」列表
      → 提取标题、截止时间、分数、完成状态
    → 发送数据到后台 Service Worker
      → 合并存储 → 更新扩展徽章
→ 点击扩展图标 → Popup 展示作业清单
```
> 📖 更详细的工作原理（爬取策略、数据流、组件通信、协调规则）→ 参见 [architecture.md](.claude/logs/architecture.md)

### 技术栈

| 技术 | 用途 |
|------|------|
| Manifest V3 | Chrome 扩展最新标准 |
| 原生 JavaScript | 零框架，轻量高效 |
| chrome.storage.local | 本地持久化存储 |
| MutationObserver | 监听 SPA 动态内容加载 |
| 三层 URL 观察 | hashchange + history API 拦截 + 轮询 |
| selectors.json | 外置选择器配置，网页改版可快速修复 |

### 项目结构

```
MOOC_reminder/
├── manifest.json           # 扩展清单
├── src/
│   ├── background/         # 后台 Service Worker（alarms、协调、徽章）
│   ├── content/            # 页面注入脚本（爬取逻辑）
│   ├── popup/              # 弹出窗口（HTML/CSS/JS）
│   └── shared/             # 共享模块（数据模型、存储、日期解析）
└── README.md
```
